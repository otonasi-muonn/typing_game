# タイピングゲームの機能拡張 (SPS化・LocalStorage問題管理UIの実装)

本ドキュメントは、タイピングゲームの打鍵速度指標の秒間化、ブラウザ内での問題追加・削除（LocalStorage永続化）機能、隠し設定モーダル、およびバリデーション・警告機能の実装内容をまとめた開発記録です。

## 実施した主な変更内容

### 1. 秒間打鍵数 (SPS: Strokes Per Second) への移行
* **変更理由**: 従来のWPM（分間打鍵数）表記から、より直感的で高精度な秒間打鍵数（SPS）での評価を行うため。
* **実装内容**:
  * ゲーム結果画面（リザルト）のWPMカードを「秒間打鍵数 (SPS)」に変更し、IDを `res-sps` に更新しました。
  * ゲーム終了時（`GameManager.endGame()`）に、総正解打鍵数を経過秒数で除算する計算ロジック（`correctCount / elapsedTime`）を実装。
  * 表示を小数点以下2桁の数値で統一し、SPSのリアルタイム性と信頼性を向上させました。

### 2. 隠し設定モーダルUIの実装 (HTML / CSS)
* **変更理由**: プレイヤーが誤って設定画面を開かないよう、通常のプレイ画面からは分かりにくい位置に設定導線を配置するため。
* **実装内容**:
  * スタート画面の右下隅に、薄い半透明（ホバー時のみ明るくなる）の⚙ボタンを絶対配置しました。
  * ⚙ボタンクリックで、ダークネオン調の管理者用設定モーダルを表示する仕組みを構築。
  * CSSの `backdrop-filter` や `box-shadow` を駆使し、サイバーパンクなデザインテーマと調和する高級感のあるモーダルUIをデザインしました。

### 3. LocalStorage を用いた問題管理機能
* **変更理由**: GitHub Pagesなどの静的ホスティングサーバー上でも、各ユーザーが独自に問題を登録・削除でき、リロード後もデータが維持されるようにするため。
* **実装内容**:
  * `QuestionRepository` クラスに LocalStorage 連携ロジックを実装。
  * 初回ロード時は、LocalStorage ➔ `questions.json` ➔ `FALLBACK_QUESTIONS` (コード内ハードコード) の優先順位で読み込み、自動的に LocalStorage へ保存。
  * 追加された問題は JavaScript 側で濁点コードを標準形式（NFC）に正規化し、OS間の差異による判定不具合を防ぎます。

### 4. 設定バリデーションと警告システム
* **実装内容**:
  * **よみがな入力バリデーション**: よみがなの入力に対し、ひらがな・長音記号（ー）・読点・感嘆符・疑問符（、。？！）のみを許可する正規表現バリデーション（`/^[ぁ-んー、。？！\?\!]+$/`）を導入。漢字やアルファベット、カタカナの誤入力を検知し、適切な警告を画面上に表示します。
  * **出題問題数変更**: 設定モーダルから出題問題数（初期値10）を可変に変更可能に。
  * **重複警告表示**: 登録問題数が設定された出題数を下回る場合、ゲーム実行時に問題が重複して選ばれる可能性があることを示す赤文字の警告をモーダル内で動的に切り替えるロジックを構築しました。
